## 為什麼要執行此變更

建立一個「對話即文件」的平台，將技術決策過程與 Debug 紀錄轉化為高品質的技術文章。這能減少手動撰寫文件的摩擦，並實時捕捉工程工作的「為什麼」與「如何做」，遵循 QPQ (quidproquo.cc) 的技術棧與哲學。

## 變更內容

- **核心平台**：全新的 Astro 專案架構，針對 Cloudflare Pages/Workers/D1 進行優化。
- **AI 優先設計**：整合 `llms.txt`、JSON-LD 與結構化元數據，便於 AI 代理人檢索。
- **雙語支持**：基礎 i18n 架構，支援繁體中文與英文內容。
- **資訊流 UI**：極簡的「Hacker-style」動態牆，支援多維度標籤過濾。
- **內容管道**：將 Claude Code 對話紀錄轉換為結構化 Astro Markdown 文章的工作流。
- **專案展示**：專門用於展示作品與側邊專案的區塊。

## 能力 (Capabilities)

### 新增能力
- `content-engine`：管理技術文章、專案與標籤過濾功能。
- `i18n-framework`：支援雙語內容交付。
- `conversation-ingestion`：將原始對話紀錄轉換為格式化文章的管道。
- `ai-readiness`：用於 AI 代理人發現與解析的標準化結構。

### 修改能力
- 無

## 影響範圍

- **程式碼庫**：使用 Astro 的全新專案結構。
- **部署**：部署至 Cloudflare Pages 的流水線。
- **數據**：使用 Cloudflare D1 處理動態數據需求（如搜尋、數據統計）。
- **工作流**：全新的本地技術寫作工具鏈。
